# Git Workflow

## Commits

Format: `type(scope): imperative message`

| Type | Use |
|---|---|
| `feat` | New feature |
| `fix` | Bug fix |
| `refactor` | Code change without feat or fix |
| `docs` | Documentation |
| `test` | Tests |
| `chore` | Maintenance tasks |
| `perf` | Performance improvements |
| `ci` | CI/CD |
| `style` | Formatting (no logic changes) |

Rules:
- No AI attribution: no `Co-authored-by: Claude`, `[AI]`, `Generated by`.
- Message in English, lowercase, no trailing period.
- Subject ≤ 50 chars. Body only when "why" isn't obvious.
- One commit per logical change. No "WIP" commits on shared branches.

## Branches

- `feature/<slug>` — new features
- `fix/<slug>` — bug fixes
- `refactor/<slug>` — refactors without functional change
- `chore/<slug>` — maintenance

## PRs

- Title: same rules as commits.
- Description: what + why + test plan.
- Reviewers assigned. No self-merge without review.
- Squash merge to main. Keep history clean.
