# Git Workflow

## Commits

Formato: `tipo(scope): mensaje en imperativo`

| Tipo | Uso |
|---|---|
| `feat` | Nueva funcionalidad |
| `fix` | Corrección de bug |
| `refactor` | Cambio de código sin feat ni fix |
| `docs` | Documentación |
| `test` | Tests |
| `chore` | Tareas de mantenimiento |
| `perf` | Mejoras de rendimiento |
| `ci` | CI/CD |
| `style` | Formato (no cambios de lógica) |

Reglas:
- Sin atribución de AI: nada de `Co-authored-by: Claude`, `[AI]`, `Generated by`.
- Mensaje en inglés, lowercase, sin punto final.
- Subject ≤ 50 caracteres. Body solo si el "why" no es obvio.
- Un commit por cambio lógico. No commits "WIP" en branches compartidos.

## Branches

- `feature/<slug>` — features nuevas
- `fix/<slug>` — correcciones
- `refactor/<slug>` — refactors sin cambio funcional
- `chore/<slug>` — mantenimiento

## PRs

- Título: misma regla que commits.
- Descripción: what + why + test plan.
- Reviewers asignados. Sin self-merge sin review.
- Squash merge a main. Mantener historial limpio.
